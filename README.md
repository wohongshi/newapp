# 智能收银台 (Smart POS)

基于 Flutter + Rust 的本地收银系统，纯离线运行，无需网络。

## ✨ 功能特性

### 📱 条码扫描
- 手机摄像头实时扫描商品条形码
- 扫码自动识别商品并加入购物车
- 未识别条码可快速新建商品

### 🛒 收银结账
- 购物车管理（增删改查）
- 自动计算价格和优惠
- 多种支付方式记录
- 交易历史记录

### 📏 定价规则引擎
- 灵活的条件-动作规则系统
- 支持多种条件组合（数量、品牌、价格、总金额等）
- 支持多种动作（固定价格、折扣、立减、买一送一）
- 规则优先级排序
- 可随时启用/禁用规则

### 📊 收益分析
- 今日/本周/本月/今年收益统计
- 收入趋势图表
- 品牌分布饼图
- 利润率分析
- 交易明细查看

### 📦 商品管理
- 按品牌/价格自动分类
- 搜索和筛选
- 库存管理
- 成本和售价管理

### 💾 备份恢复
- JSON格式全量备份
- 一键恢复数据
- CSV导出商品数据

### 🔒 安全
- 应用密码保护（4-8位数字PIN）
- 首次使用可选设置
- 本地安全存储

## 🏗 技术架构

- **前端**: Flutter (Dart) - Material Design 3
- **后端逻辑**: Rust - 高性能计算引擎
- **数据库**: SQLite (sqflite)
- **安全存储**: flutter_secure_storage

## 📱 构建

项目使用 GitHub Actions 自动构建，推送到 `main` 分支即可触发构建。

### 本地构建

```bash
# 1. 安装依赖
flutter pub get

# 2. 编译Rust后端
cd rust_backend
cargo ndk -t arm64-v8a --platform 21 -- build --release
mkdir -p ../android/app/src/main/jniLibs/arm64-v8a
cp target/aarch64-linux-android/release/libnewapp_backend.so ../android/app/src/main/jniLibs/arm64-v8a/
cd ..

# 3. 构建APK
flutter build apk --release
```

## 📋 定价规则编写规范

### 规则结构
每条规则由以下部分组成：
- **规则名称**: 简短描述规则用途
- **优先级**: 数字越大越先执行 (0-10)
- **条件(Conditions)**: 触发规则的前置条件
- **动作(Action)**: 满足条件后的价格调整方式

### 条件类型
| 字段 | 说明 | 示例 |
|------|------|------|
| quantity | 单件购买数量 | 数量 ≥ 20 |
| total_quantity | 购物车总数量 | 总数量 ≥ 50 |
| brand | 品牌名称 | 品牌 = 可口可乐 |
| price | 商品原价 | 单价 > 5 |
| total_amount | 购物车总金额 | 总金额 ≥ 100 |

### 条件运算符
| 运算符 | 含义 |
|--------|------|
| gte | ≥ 大于等于 |
| lte | ≤ 小于等于 |
| eq | = 等于 |
| neq | ≠ 不等于 |
| gt | > 大于 |
| lt | < 小于 |
| contains | 包含 |

### 动作类型
| 类型 | 说明 | 值含义 |
|------|------|--------|
| fixed_price | 固定价格 | 目标单价(元) |
| percent_discount | 百分比折扣 | 折扣百分比 |
| amount_discount | 立减金额 | 每件减免(元) |
| bogo | 买一送一 | 无需填值 |

### 作用范围
| 范围 | 说明 |
|------|------|
| item | 仅对单个商品生效 |
| all_matching | 对所有满足条件的商品生效 |
| cart | 对整个购物车按比例分摊 |

### 示例

**示例1**: 0.8元商品满20件价格调整为0.75元
- 条件: 单价 = 0.8 且 数量 ≥ 20
- 动作: 固定价格 0.75（单品）

**示例2**: 品牌A全场9折
- 条件: 品牌 = A
- 动作: 百分比折扣 10%（所有匹配商品）

**示例3**: 满100元整单减5元
- 条件: 总金额 ≥ 100
- 动作: 立减 5（整单）

**示例4**: 买一送一促销
- 条件: (无条件或指定品牌)
- 动作: 买一送一（单品）
